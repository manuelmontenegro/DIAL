Sintaxis de fórmulas

P ::= At                // fórmula atómica
    | \(\neg\)P                // negación
    | P \({\wedge}\) Q             // conjunción
    | P \({\vee}\) Q             // disjunción
    | P ==> Q           // implicación
    | P <=> Q           // equivalencia
    | (\(\forall\)i : P : Q)      // cuantificador universal
    | (\(\exists\)i : P : Q)      // cuantificador existencial

  • La notación (\(\forall\)i : P : Q) significa \(\forall i : P \Rightarrow Q\)
  • La notación (\(\exists\)i : P : Q) significa \(\exists i : P \wedge Q\)

  • Utilizamos P, Q, R para denotar fórmulas.

  • Fórmula lógica puede ser, o bien una fórmula atómica (enseguida veremos qué significa eso)

  • se cumple P y se cumple Q

  • obien se cumple P o bien se cumple Q, o bien ambas

  • El cuantificador universal, aunque el concepto es conocido, la forma de expresarlo va a ser como véis aquí.

  • El predicado P se utiliza para el rango de la variable i

como esta sintaxis es relativamente poco frecuente, paso a comentar lo que significa.

Expresiones cuantificadas

(Σi : P(i) : e(i))

Suma de todos los e(i) que satisfacen P(i)

(Σi : 1 <= i <= 5 : i)

(Σi : 0 <= i < 3 : i2)

(Πi : P(i) : e(i))

Producto de todos los e(i) que satisfacen P(i)

(#i : P(i) : Q(i))

Número de valores de i para los que se cumple P(i) \({\wedge}\) Q(i)

(#i : 0 <= i < n : V[i] == 0)

`(SUM i : 1 <= i <= 5 : i)`{.pseudo} equivale a `1 + 2 + 3 + 4 + 5 = 15`{.pseudo}

`(SUM i : 0 <= i < 3 : i^2^)`{.pseudo} equivale a `0^2^ + 1^2^ + 2^2^ = 5`{.pseudo}

`(#i : 0 <= i < n : V[i] == 0)`{.pseudo} indica el número de posiciones del vector `V`{.pseudo} que contienen el valor 0, suponiendo que la longitud del vector `V`{.pseudo} es `n`{.pseudo}.
(máx i : P(i) : e(i))

Máximo de todos los e(i) tales que se cumple P(i).

(máx i : -10 <= i <= 10 \({\wedge}\) i2 == 9 : i)

(máx i : 0 <= i < n : V[i])

(mín i : P(i) : e(i))

Mínimo de todos los e(i) tales que se cumple P(i).

(mín i : -10 <= i <= 10 \({\wedge}\) i2 == 9 : i)

(mín i : -5 <= i <= 5 : i2)

`(máx i : -10 <= i <= 10 && i^2^ == 9 : i)`{.pseudo} es igual a 3.

`(máx i : 0 <= i < n : V[i])`{.pseudo} denota el valor máximo de un vector `V`{.pseudo}, suponiendo que tiene longitud `n`{.pseudo}.

`(mín i : -10 <= i <= 10 && i^2^ == 9 : i)`{.pseudo} es igual a -3.

`(mín i : -5 <= i <= 5 : i^2^)`{.pseudo} es igual a 0.

  • Admitimos expresiones cuantificadas con varias variables:

    (máx i, j : -5 <= i <= 0 \({\wedge}\) -2 <= j <= 3 : i * j)

    (Σi, j : 0 <= i < j < n : V[i] * V[j])

A veces va a ser más cómodo tener varias variables cuantificadas. Esto equivale a tener cuantificadores anidados.

Máximo producto que se puede formar con un producto de i * j donde la i puede estar entre -5 y 0 y la j entre -2 y 3. El máximo es -5 * -2 = 10

Sobre los tipos

  • Supondremos que cada variable tiene un tipo: entero, real, booleano o array.

  • No admitimos fórmulas que contengan discrepancias entre los tipos.

    Por ejemplo, si x es de tipo entero y V es de tipo array, no permitimos expresiones como las siguientes:

    V + 3               x[1]            x + true

Un pequeño apunte. Aunque no hemos definido formalmente un sistema de tipos, vamos a suponer que cada variable lógica tiene un tipo.

Y no vamos a permitir fórmulas que estén mal formadas con respecto a los tipos.

Variables libres y ligadas

Variables ligadas

  • En las expresiones y fórmulas cuantificadas:

    (\(\forall\)i : P : Q)
    (\(\exists\)i : P : Q)
    (Σi : P : e)
    …

    decimos que las apariciones de la variable i dentro de P y Q están ligadas por el cuantificador correspondiente.

  • Por ejemplo: (\(\forall\)i : 0 <= i < 10 : V[i] <= n)

La variable i está ligada por el cuantificador \(\forall\), pero las variables V y n no están ligadas por ningún cuantificador.

Variables libres

  • Decimos que una aparición de una variable es libre si no está ligada por ningún cuantificador.

    (\(\forall\)i : 0 <= i < 10 : V[i] <= n)

    (\(\exists\)j : 0 <= j < 10 : j * 2 == x)

    (\(\forall\)i : 0 <= i < j : (\(\exists\)j : 0 <= j < i : j2 == i))

Fijaos que j aparece ligada y libre en la misma fórmula. Como esto se presta mucho a confusión.

Renombramiento de variables ligadas

  • En una expresión o fórmula cuantificada (\(\forall\)i : P : Q) siempre es posible renombrar la variable i por otra variable que no aparezca en P o Q, sin alterar la semántica de la expresión o fórmula.

  • (Σi : 0 <= i < x : i) equivale a (Σj : 0 <= j < x : j).

  • (\(\forall\)i : 0 <= i < j : (\(\exists\)j : 0 <= j < i : j2 == i))

    equivale a

    (\(\forall\)i : 0 <= i < j : (\(\exists\)k : 0 <= k < i : k2 == i))

Sustitución de variables

Sustitución de variables

  • Sean e, e’ dos expresiones y x una variable.

  • Denotamos por e[x / e’] el resultado de reemplazar en e todas las apariciones libres de x por la expresión e’.

  • Ejemplos:

    (x + 2)[x / 3] == 3 + 2

    (V[i])[i / y * 3] == V[y * 3]

    (Σi : 0 <= i < n : V[i])[i / z] == (Σi : 0 <= i < n : V[i])

Cuidado: una sustitución no puede aplicarse a variables ligadas

  • Supondremos que ninguna de las variables de e’ aparece ligada en e.

    • De lo contrario, renombramos previamente las variables ligadas en e.

  • Ejemplo:

      (Σi : 0 <= i < n : V[i])[n / i + 1]
    ==
      (Σj : 0 <= j < n : V[j])[n / i + 1]
    ==
      (Σj : 0 <= j < i + 1 : V[j])

Sustitución simultánea

  • Denotamos por

    e[x1 / e1, …, xn / en]

    el resultado de reemplazar en e todas las apariciones libres de cada xi por su correspondiente ei.

  • La sustitución de todas las variables xi se realiza de manera simultánea.

  • Por ejemplo:

    (x + y)[x / 4, y / z * 2] == 4 + z * 2

No es lo mismo aplicar una sustitución simultánea de dos variables

  (x + y)[x / y + 2, y / 3]
==
  (y + 2 + 3)

que aplicar dos sustituciones en secuencia

  (x + y)[x / y + 2][y / 3]
==
  (y + 2 + y)[y / 3]
==
  (3 + 2 + 3)

Ojo, que en general no es lo mismo aplicar una sustitución simultánea por dos sustituciones seguidas.

Sustitución en fórmulas lógicas

  • Del mismo modo, definimos P[x / e] como el resultado de sustituir en P las apariciones libres de x por e.

      (\(\forall\)k : 0 <= k < i : V[k] >= 0)[i / i + 1]
    ==
      (\(\forall\)k : 0 <= k < i + 1 : V[k] >= 0)

Semántica de fórmulas lógicas

Concepto de estado

  • Un estado \(\sigma\) es una función que asocia cada variable con un valor de su tipo correspondiente.

  • Representamos los estados mediante \(\sigma\), \(\sigma_1\), \(\sigma'\), etc.

  • Notación: \([x_1 \mapsto v_1, \dots, x_n \mapsto v_n]\) denota el estado \(\sigma\) tal que:

\[ \text{Para todo }x: \qquad \sigma(x) = \begin{cases} v_1 & \text{si }x = x_1\\ \qquad \vdots \\ v_n & \text{si }x = x_n\\ 0 & \text{e.o.c. }\\ \end{cases} \]

Semántica de fórmulas lógicas

  • Decimos que un estado \(\sigma\) satisface una fórmula lógica \(P\)   ( \(\sigma \vDash P\) ) si la fórmula es cierta al sustituir cada variable libre \(x\) de \(P\)  por su valor \(\sigma(x)\).

  • Ejemplos:

    • Sea \(\sigma = [\texttt{x} \mapsto 3, \texttt{y} \mapsto 5]\)

      \(\sigma \vDash\) x > 0 \({\wedge}\) x < 5

      \(\sigma \vDash\) x < y

      \(\sigma \not\vDash\) y < 0

    • Sea \(\sigma = [\texttt{V} \mapsto [5, 3, 5, -3], \texttt{n} \mapsto 4]\)

      \(\sigma \vDash\) (#i : 0 <= i < n : V[i] == 5) > 1

      \(\sigma \not\vDash\) (\(\forall\)i : 0 <= i < n : V[i] >= 0)

Casos extremos: rango vacío en cuantificadores

  • (\(\forall\)i : P(i) : Q(i))

    Si P(i) no se satisface para ningún valor de i, la fórmula es cierta.

  • (\(\exists\)i : P(i) : Q(i))

    Si P(i) no se satisface para ningún valor de i, la fórmula es falsa.

  • (Σi : P(i) : e(i))

    Si P(i) no se satisface para ningún valor de i, el sumatorio es 0.

  • (Πi : P(i) : e(i))

    Si P(i) no se satisface para ningún valor de i, el producto es 1.

  • (mín i : P(i) : e(i)), (máx i : P(i) : e(i))

    Exigimos que exista un valor de i satisfaga P(i).

Validez de fórmulas lógicas

  • Decimos que una fórmula \(P\) es válida si \(\sigma \vDash P\) para cualquier estado \(\sigma\).

  • Ejemplos:

    • No es válida: x > 0 \({\wedge}\) x < 5

    • Sí es válida: x + z >= y + z ==> x >= y

Equivalencia de fórmulas lógicas

  • Dos fórmulas \(P\) y \(Q\) son lógicamente equivalentes (\(P \equiv Q\)) si se cumple

    \[ \sigma \vDash P \Longleftrightarrow \sigma \vDash Q \]

    para todo estado \(\sigma\).

  • Por ejemplo, x > 0 y \(\neg\)(x <= 0) son lógicamente equivalentes.

Leyes de equivalencia lógica

Conmutatividad, asociatividad e idempotencia

P \({\wedge}\) Q === Q \({\wedge}\) P
(P \({\wedge}\) Q) \({\wedge}\) R === P \({\wedge}\) (Q \({\wedge}\) R)
P \({\wedge}\) P === P

P \({\vee}\) Q === Q \({\vee}\) P
(P \({\vee}\) Q) \({\vee}\) R === P \({\vee}\) (Q \({\vee}\) R)
P \({\vee}\) P === P

Elemento neutro

P \({\wedge}\) true === P

P \({\vee}\) false === P

Distributividad

P \({\wedge}\) (Q \({\vee}\) R) === (P \({\wedge}\) Q) \({\vee}\) (P \({\wedge}\) R)

P \({\vee}\) (Q \({\wedge}\) R) === (P \({\vee}\) Q) \({\wedge}\) (P \({\vee}\) R)

Negación y De Morgan

\(\neg\)\(\neg\)P === P

P \({\wedge}\) \(\neg\)P === false
P \({\vee}\) \(\neg\)P === true

\(\neg\)(P \({\wedge}\) Q) === \(\neg\)P \({\vee}\) \(\neg\)Q
\(\neg\)(P \({\vee}\) Q) === \(\neg\)P \({\wedge}\) \(\neg\)Q

Implicación simple y doble

P ==> Q === \(\neg\)P \({\vee}\) Q
P ==> (Q ==> R) === P \({\wedge}\) Q ==> R

P <=> Q === (P ==> Q) \({\wedge}\) (Q ==> P)

Gran De Morgan

\(\neg\)(\(\forall\)i : P : Q) === (\(\exists\)i : P : \(\neg\)Q)

\(\neg\)(\(\exists\)i : P : Q) === (\(\forall\)i : true : \(\neg\)P \({\vee}\) \(\neg\)Q)

Renombramiento

(\(\forall\)i : P : Q) === (\(\forall\)j : P[i / j] : Q[i / j])
(\(\exists\)i : P : Q) === (\(\forall\)j : P[i / j] : Q[i / j])

si j no aparece ni en P ni en Q
