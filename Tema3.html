3.1Conceptos básicos

Funciones recursivas

  • Una función recursiva es aquella que se llama a sí misma directa o indirectamente para resolver un problema.

Recursión directa

method f(x)
{
  …
  f(x - 1); Llamada recursiva
  …
}

Recursión indirecta

method f(x)
{
  … g(x); …
}

method g(z)
{
  …
  f(z / 2);  Llamada recursiva a f
  …
}

Partes de una función recursiva

  1. Caso base:
    • Es la condición que detiene la recursión.
    • Caso suficientemente sencillo de resolver de manera directa.
  2. Caso recursivo:
    • Es el caso en el que la función se llama a sí misma.
    • Construye la solución basándose en la solución de un subproblema más pequeño.

method factorial(n: int) returns (f: int)
{
  if n == 0 {
    f := 1;                   Caso base
  } else {
    g := factorial(n - 1);    Caso recursivo
    f := g * n;
  }
}

Recursión simple y recursión múltiple

Recursión simple y recursión múltiple

  • Simple: cada llamada da lugar, como mucho, a una única llamada recursiva.
  • Múltiple: cada llamada puede dar lugar a dos o más llamadas recursivas.

Recursión simple

method suma(n: int) returns (s: int)
{
  if n == 0 {
    s := 1;
  } else {
    s1 := suma(n - 1);
    s := s1 + n;
  }
}

Recursión múltiple

method fib(n: int) returns (f: int)
{
  if n == 0 \({\vee}\) n == 1 {
    f := n;
  } else {
    f0 := fib(n - 1);
    f1 := fib(n - 2);
    f := f0 + f1;
  }
}

Recursión simple y recursión múltiple

  • ¡Ojo! La siguiente función es recursiva simple.

    method gcd(a: int, b: int) returns (g: int)
    {
      if a == b {
        g := a;
      } else if a > b {
        g := gcd(a - b, b);
      } else {
        g := gcd(a, b - a);
      }
    }

Recursión final y no final

Recursión simple final

  • La llamada recursiva es la última operación en cada caso recursivo.
  • El resultado de la llamada recursiva se devuelve directamente, sin realizar cálculos sobre él.

method resto(a: int, b: int) returns (r: int)
{
  if a < b {
    r := a;
  } else {
    r := resto(a - b, b);
  }
}

Recursión simple no final

  • La llamada recursiva no es la última operación realizada en los casos recursivos.
    • Si hay varios casos recursivos, basta con que esta condición se cumpla en uno de ellos.
  • Es decir, tras la llamada recursiva, hay operaciones pendientes que deben aplicarse al resultado de esta llamada.

method factorial(n: int) returns (f: int)
{
  if n == 0 {
    f := 1;
  } else {
    f1 := factorial(n - 1);
    f := f1 * n;
  }
}

La pila de llamadas
(call stack)

El precio de la recursión

  • Durante la ejecución de un programa se almacena una pila de llamadas a función (call stack).

  • Por ejemplo, dado el siguiente conjunto de funciones:

method m() returns (r: int)       { r := h(3); }
method h(x: int) returns (r: int) { r := g(x + 1) * 2; }
method g(x: int) returns (r: int) { r := f(x) - 3; }
method f(x: int) returns (r: int) { r := x + 1; }

supongamos la llamada m()

Pila de llamadas

  • Al ejecutar la llamada m(), tenemos la siguiente pila de llamadas en el momento de ejecutarse la llamada a f:

    • En la cima de la pila está la función que se está ejecutando actualmente (f).
    • En la subcima está la función que llamó a f, y así sucesivamente.
 
f(4) Función ejecutándose actualmente
g(4) Función que llama a f
h(3) Función que llama a g
m() Llamada inicial

Pila de llamadas

 
f(4)
g(4)
h(3)
m()
  • A cada elemento de la pila se le llama frame.
  • Cada frame contiene los valores de los parámetros y definiciones locales correspondientes a esa llamada.
  • Cada vez que se llama a una función se inserta un nuevo frame en la pila.
  • Cuando una función finaliza, se elimina su frame de la pila
  • Ejemplo: ejecución de factorial(4):
 
factorial(0)
factorial(1)
factorial(2)
factorial(3)
factorial(4)

 

  • El tamaño de la pila de llamadas tiene un límite.
  • Si la cadena de llamadas es muy grande, se puede superar ese límite de tamaño, provocando un desbordamiento de pila (Stack Overflow).

 

Sin embargo, esto no pasa con las funciones recursivas finales.

Optimización de recursión final

  • Si la función es recursiva final, cada frame puede sustituir al anterior.

  • El tamaño de la pila se mantiene constante, por lo que la función puede hacer un número ilimitado de llamadas recursivas sin que haya desbordamiento de pila.

 
 
 
 
 
resto(16, 3)

➤

➤

Árbol de activación

Árbol de activación

  • Es un árbol que representa gráficamente las llamadas recursivas que se realizan a partir de una llamada inicial.

    • Raíz: Llamada inicial

    • Hojas: Casos base

    • Nodos internos: Casos recursivos

Árbol de activación: ejemplo

method fib(n: int) returns (f: int)
{
  if n == 0 \({\vee}\) n == 1 {
    f := n;
  } else {
    f0 := fib(n - 1);
    f1 := fib(n - 2);
    f := f0 + f1;
  }
}  

Árbol de activación: recursión simple

  • En el caso de funciones recursivas simples, el árbol de activación tiene una sola rama.

method factorial(n: int) returns (f: int)
{
  if n == 0 {
    f := 1;
  } else {
    f1 := factorial(n - 1);
    f := f1 * n;
  }
}

Corrección total de funciones recursivas

  • Para poder demostrar la corrección total hemos de asegurarnos de que la recursión termine para cualquier valor del parámetro que satisfaga la precondición.

  • Para ello nos basamos en las funciones de cota.

Recordatorio

Una función de cota de un bucle es una expresión que depende de las variables del bucle y que cumple las siguientes condiciones:

  1. Mantiene un valor ≥ 0 antes de cada iteración del bucle.

  2. Decrece estrictamente cada vez que realizamos una iteración del bucle.

Funciones de cota en recursión

Una función de cota de un bucle de una función recursiva es una expresión que depende de las variables del bucle los parámetros de la función y que cumple las siguientes condiciones:

  1. Mantiene un valor ≥ 0 antes de cada iteración del bucle en cada llamada recursiva.

  2. Decrece estrictamente cada vez que realizamos una iteración del bucle una llamada recursiva.

Regla de verificación de llamadas recursivas

Suponiendo que t es la función de cota que depende de x1, …, xn:

method f(x1: tipo1, …, xn: tipon) returns (y1: tipo1, …, ym: tipon)
  requires P
  ensures Q
  decreases t

Se debe demostrar, en primer lugar, que P ==> t >= 0.

Verificación de funciones recursivas simples

Esquema básico de función recursiva simple

method f(x̅) returns (y̅)
  requires P
  ensures Q
  decreases t
{
  if b then {
    y̅ := triv(x̅)
  } else {
    z̅ := f(s(x̅));
    y̅ := c(z̅, x̅)
  }
}

triv(x̅)

Expresión que denota el resultado del caso base.

s(x̅)

Expresión que denota los argumentos pasados a la llamada recursiva.

c(z̅, x̅)

Expresión que denota las transformaciones sobre el resultado de la llamada recursiva.

Si f es recursiva final, c(z̅, x̅) = z̅.

Ejemplo

method factorial(x: int) returns (y: int)
  requires x >= 0
  ensures y == x!
  decreases x
{
  var z: int;
  if x == 0 then {
    y := 1
  } else {
    z := factorial(x - 1);
    y := z * x
  }
}

triv(x) = 1

s(x) = x - 1

c(z, x) = z * x

Condiciones de verificación (1): caso base

method f(x̅) returns (y̅)
  requires P
  ensures Q
  decreases t
{
  if b then {
    y̅ := triv(x̅)
  } else {
    z̅ := f(s(x̅));
    y̅ := c(z̅)
  }
}

  1. La condición del caso base está bien definida:

    P ==> def(b)

  2. El caso base es correcto:

    P \({\wedge}\) b ==> Q[y̅/triv(x̅)]

Condiciones de verificación (2): caso recursivo

method f(x̅) returns (y̅)
  requires P
  ensures Q
  decreases t
{
  if b then {
    y̅ := triv(x̅)
  } else {
    z̅ := f(s(x̅));
    y̅ := c(z̅)
  }
}

  1. Se cumple la precondición en la llamada recursiva:

    P \({\wedge}\) \(\neg\)b ==> P[x̅/s(x̅)]

  2. Se cumple la postcondición a partir del resultado de la llamada recursiva:

    P \({\wedge}\) \(\neg\)b \({\wedge}\) Q[x̅/s(x̅), y̅/z̅] ==> Q[y̅/c(z̅, x̅)]

Condiciones de verificación (3): terminación

method f(x̅) returns (y̅)
  requires P
  ensures Q
  decreases t
{
  if b then {
    y̅ := triv(x̅)
  } else {
    z̅ := f(s(x̅));
    y̅ := c(z̅)
  }
}

  1. La función de cota es ≥ 0

    P ==> t >= 0

  2. La función de cota decrece en cada llamada recursiva:

    P \({\wedge}\) \(\neg\)b ==> t[x̅/s(x̅)] < t

Ejemplo: factorial

method factorial(x: int) returns (y: int)
  requires x >= 0
  ensures y == x!
  decreases x
{
  var z: int;
  if x == 0 then {
    y := 1
  } else {
    z := factorial(x - 1);
    y := z * x
  }
}

  1. Condición de caso base bien definida:

    x >= 0 ==> true

  2. Caso base correcto:

    x >= 0 \({\wedge}\) x == 0 ==> 1 == x!

  3. Precondición caso recursivo:

    x >= 0 \({\wedge}\) x != 0 ==> x - 1 >= 0

  4. Postcondición en caso recursivo:

    x >= 0 \({\wedge}\) x != 0 \({\wedge}\) z == (x - 1)!
       ==> z * x == x!

  2. Función de decrece en llamadas recursivas:

    x >= 0 \({\wedge}\) x != 0 ==> x - 1 < x

3.3Análisis de coste de algoritmos recursivos

Recurrencia de coste

  • A partir de una función recursiva puede obtenerse una función de coste \(T(n_1, \ldots, n_m)\) que depende de los tamaños tamaño \(n_1, \ldots, n_m\) de la entrada.

  • Esta función \(T\) se expresa fácilmente mediante una recurrencia.

method factorial(x: int) returns (y: int)
  requires x >= 0
  ensures y == x!
  decreases x  
{
  var z: int;
  if x == 0 then {
    y := 1
  } else {
    z := factorial(x - 1);
    y := z * x
  }
}

Si \(n\) es el tamaño de la entrada (el número x):

\(T(n) = \begin{cases} k_1 & \text{si } n = 0 \\ k_2 + T(n-1) & \text{si } n > 0 \\ \end{cases}\)

  • \(k_1\): coste del caso base.
  • \(T(n-1)\): coste de la llamada recursiva factorial(x - 1).
  • \(k_2\): coste del resto del caso recursivo (es decir, excluyendo la llamada recursiva).

Problema

\[T(n) = \begin{cases} k_1 & \text{si } n = 0 \\ k_2 + T(n-1) & \text{si } n > 0 \\ \end{cases}\]

  • La recurrencia no nos da una idea intuitiva del orden de complejidad de \(T\).

    • ¿Es \(T(n) \in \Theta(n)\)? ¿o quizás \(T(n) \in \Theta(n^2)\)?

  • Nuestro objetivo es encontrar una forma explícita (no recursiva) de \(T\), o al menos, determinar su orden de complejidad.

¿Cómo calcular el orden de complejidad de \(T\)?

Existen dos métodos:

  • Método de los desplegados

    Laborioso, pero puede aplicarse en la mayoría de casos.

  • Reglas de la resta y de la división

    Más sencillas de aplicar, pero son aplicables a un menor número de casos.

Método de los desplegados

Método de los desplegados

  • Consiste en desplegar (esto es, «desenrollar») las llamadas recursivas, y encontrar un patrón en la fórmula resultante tras el \(i\)-ésimo desplegado.

  • Cada vez que desplegamos cada llamada recursiva, lo indicamos con un símbolo \(\stackrel{\small i}{=}\), donde \(i\) indica el número de desplegado.

Ejemplo

\[T(n) = \begin{cases} k_1 & \text{si } n = 0 \\ k_2 + T(n-1) & \text{si } n > 0 \\ \end{cases}\]

Desplegamos la definición de \(T(n)\):

\[ \begin{array}{rclcl} T(n) & \stackrel{\small 1}{=} & k_2 + T(n - 1) \\ & \stackrel{\small 2}{=} & k_2 + (k_2 + T(n - 2)) & = & 2k_2 + T(n - 2)\\ & \stackrel{\small 3}{=} & 2k_2 + (k_2 + T(n - 3)) & = & 3k_2 + T(n - 3)\\ & \stackrel{\small 4}{=} & \dots \end{array} \]

Hemos encontrado un patrón: \(T(n) \stackrel{\small i}{=} ik_2 + T(n - i)\)

Ejemplo

\[T(n) \stackrel{\small i}{=} ik_2 + T(n - i)\]

  • ¿Cuándo se llega al caso base?

    Cuando \(n - i = 0\), o equivalentemente, cuando \(i = n\). Es decir, en el \(n\)-ésimo desplegado.

    \[T(n) \stackrel{\small n}{=} nk_2 + T(0) = nk_2 + k_1\]

  • Por tanto, tenemos que \(T(n) = nk_2 + k_1\).

    • Para asegurarse, tendría que demostrarse por inducción sobre \(n\) 🤔

  • Por tanto, \(T(n) \in \Theta(n)\)

Reglas de la resta y de la división

Regla de la resta

  • Se utiliza cuando la función de coste decrece de manera constante (\(n-1\), \(n-2\), etc.) en cada llamada recursiva. Es decir, recurrencias de la forma:

    \[T(n) = \begin{cases} k_1n^d & \text{si }0 \leq n < b \\ aT(n - b) + k_2n^d & \text{si }n \geq b \\ \end{cases}\]

  • En este caso:

    \[T(n) \in \begin{cases} \Theta(n^{d+1}) & \text{si }a = 1\\ \Theta(a^{n~\text{div}~b}) & \text{si }a > 1\\ \end{cases}\]

Ejemplo

\[T(n) = \begin{cases} k_1 & \text{si } n = 0 \\ k_2 + T(n-1) & \text{si } n > 0 \\ \end{cases}\]

En este caso, \(a = 1\), \(b = 1\) y \(d = 0\). Por tanto, \(T(n) \in \Theta(n^{d+1}) = \Theta(n)\).

Regla de la división

  • Se utiliza cuando la función de coste decrece en una fracción (\(\frac{n}{2}\), \(\frac{n}{3}\), etc.) en cada llamada recursiva. Es decir:

    \[T(n) = \begin{cases} k_1n^d & \text{si }1 \leq n < b \\ aT\left(\frac{n}{b}\right) + k_2n^d & \text{si }n \geq b \\ \end{cases}\]

  • En este caso:

    \[T(n) \in \begin{cases} \Theta(n^{d}) & \text{si }a < b^d\\ \Theta(n^{d} \log~n) & \text{si }a = b^d\\ \Theta(n^{\log_b~a}) & \text{si }a > b^d\\ \end{cases}\]

3.4Ejemplo: cálculo de potencias

Cálculo de potencias en tiempo lineal

method potencia(x: int, y: int)
                        returns (p: int)
  requires y >= 0
  ensures p == xy
  decreases y
{
  var z: int;
  if y == 0 then {
    p := 1;
  } else {
    z := potencia(x, y - 1);
    p := x * z;
  }
}

  • La corrección se basa en el siguiente hecho:

    xy == x * xy-1

    para y > 0.

  • Coste en tiempo: Θ(y), utilizando la regla de la resta.

Cálculo de potencias en tiempo logarítmico

Cálculo de potencias en tiempo logarítmico

Podemos aprovechar las siguientes propiedades de las potencias:

x2n == xn * xn

x2n+1 == xn * xn * x

Dicho de otra forma:

xn == xn/2 * xn/2       // si n es par

xn == x⌊n/2⌋ * x⌊n/2⌋ * x   // si n es impar

Cálculo de potencias en tiempo logarítmico

method potencia(x: int, y: int) returns (p: int)
  requires y >= 0
  ensures p == xy
  decreases y
{
  var z: int;
  if y == 0 {
    p := 1;
  } else if y mod 2 == 0 {                
    z := potencia(x, y div 2);
    p := z * z;
  } else {
    z := potencia(x, y div 2);
    p := z * z * x;
  }
}

3.5Técnicas de inmersión

Diseño recursivo

El diseño recursivo de funciones se basa en el siguiente principio:

Si queremos resolver un problema P para una entrada x, podemos resolver ese mismo problema instanciado a una entrada x' más «fácil» de resolver. A partir de la solución de P sobre x' podemos construir una solución para el x original.

  • Ejemplos:

    • Cálculo de n! a partir de (n - 1)!

    • Cálculo de xy a partir de x⌊y/2⌋

En qué tipo de problemas es útil la inmersión y por qué la necesitamos en el diseño recursivo.

Problema

😭 A veces no es posible determinar una entrada x’ más sencilla con la que realizar la llamada recursiva.

 

method sumaVector(V[0..n): array<int>) returns (s: int)
  requires n >= 0
  ensures s == (Σi : 0 <= i < n : V[i])

 

method raizEntera(a: int) returns (r: int)
  requires a >= 0
  ensures r >= 0 \({\wedge}\) r2 <= a < (r + 1)2

Inmersión

  • Dada una función f, consiste en diseñar una función g más general.

    • g puede tener más parámetros o resultados que f.
    • g es más «general» en el sentido que, para ciertos valores de los nuevos parámetros de g, se comporta igual que f.

A la función g se le llama funcion inmersora.

A la función f se le llama funcion sumergida.

  • Hay dos tipos de inmersión:
    • Inmersión final: si g es recursiva final
    • Inmersión no final: si g es recursiva no final

Inmersión no final

Reemplazar constante por variable

  • Una técnica común para generalizar una función consiste en reemplazar una constante de su postcondición por un nuevo parámetro:

method sumaVector(V[0..n): array<int>) returns (s: int)
  requires n >= 0
  ensures s == (Σi : 0 <= i < n : V[i])

method sumaVectorGen(V[0..n): array<int>,  j: int ) returns (s: int)
  requires n >= 0
  ensures s == (Σi :  j  <= i < n : V[i])

Fortalecimiento de la precondición

  • La función inmersora suele tener condiciones adicionales en la precondición:

method sumaVectorGen(V[0..n): array<int>, j: int) returns (s: int)
  requires n >= 0 \({\wedge}\)  0 <= j <= n 
  ensures s == (Σi : j <= i < n : V[i])

Relación entre ambas funciones

Para j == 0:

  • La postcondición de sumaVectorGen implica la postcondición de sumaVector.

        s == (Σi : j <= i < n : V[i]) \({\wedge}\) j == 0
    ==>
        s == (Σi : 0 <= i < n : V[i])

  • La precondición de sumaVector implica la precondición de sumaVectorGen.

        n >= 0 \({\wedge}\) j == 0
    ==>
        n >= 0 \({\wedge}\) 0 <= j <= n

Esto nos permite definir sumaVector del siguiente modo:

method sumaVector(V[0..n): array<int>) returns (s: int)
  requires n >= 0
  ensures s == (Σi : 0 <= i < n : V[i])
{
  s := sumaVectorGen(V, 0);
}

Diseño de sumaVectorGen

  • Al contrario que sumaVector, la función inmersora sumaVectorGen sí tiene una descomposición obvia en un problema más sencillo.

  • A partir de (Σi : j + 1 <= i < n : V[i]) se calcula (Σi : j <= i < n : V[i]).

method sumaVectorGen(V[0..n): array<int>, j: int) returns (s: int)
  requires n >= 0 \({\wedge}\) 0 <= j <= n
  ensures s == (Σi : j <= i < n : V[i])
  decreases n - j
{
  var s’: int;
  if j == n then {
    s := 0;
  } else {
    s’ := sumaVectorGen(V, j + 1);
    s  := s’ + V[j];
  }
}

Corrección de la función inmersora

  1. Condición de caso base bien definida:

    n >= 0 \({\wedge}\) 0 <= j <= n ==> true

  2. Caso base correcto:

       n >= 0 \({\wedge}\) 0 <= j <= n \({\wedge}\) j == n
    ==>
       0 == (Σi : j <= i < n : V[i])

  3. Precondición en llamada recursiva:

       n >= 0 \({\wedge}\) 0 <= j <= n \({\wedge}\) j != n
    ==>
       n >= 0 \({\wedge}\) 0 <= j + 1 <= n

  1. Caso recursivo:

      n >= 0 \({\wedge}\) 0 <= j <= n \({\wedge}\) j != n
      \({\wedge}\) s’ == (Σi : j + 1 <= i < n : V[i])
    ==>
      s’ + V[j] == (Σi : j <= i < n : V[i])

  2. Función de cota ≥ 0:

    n >= 0 \({\wedge}\) 0 <= j <= n ==> n - j >= 0

  3. Función de cota decrece:

       n >= 0 \({\wedge}\) 0 <= j <= n \({\wedge}\) j != n
    ==>
       n - (j + 1) < n - j

Inmersión final

Inmersión final

  • Consiste en introducir parámetros o resultados adicionales para que la función inmersora sea recursiva final.

  • La postcondición de la función inmersora ha de coincidir con la de la función sumergida.

    • Para ello se han de introducir parámetros acumuladores si es necesario.

  • El caso base ha de devolver el resultado final acumulado.

  • La función inmersora no puede devolver un resultado parcial
  • La idea es que en el caso base yo ya tengo que tener el resultado de la función

Ejemplo: suma de los elementos de un vector

method sumaVector(V[0..n): array<int>) returns (s: int)
  requires n >= 0
  ensures s == (Σi : 0 <= i < n : V[i])

Reemplazando la n por un nuevo parámetro, obtenemos:

s == (Σi : 0 <= i <  j  : V[i]) \({\wedge}\)  j == n 

  • La condición j == n determina el caso base.

  • La condición s == (Σi : 0 <= i <  j  : V[i]), renombrando la variable s por otro parámetro sacum, se añade a la precondición de la función inmersora.

method sumaVectorGen(V[0..n): array<int>, j: int, sacum: int) returns (s: int)
  requires n >= 0 \({\wedge}\) 0 <= j <= n \({\wedge}\) sacum == (Σi : 0 <= i < j : V[i])
  ensures s == (Σi : 0 <= i < n : V[i])

  • Los valores iniciales de los parámetros nuevos se determinan de modo que la precondición de sumaVectorGen se satisfaga fácilmente.

    • Por ejemplo: j == 0, sacum == 0.
  • La función sumergida podría implementarse del siguiente modo:

method sumaVector(V[0..n): array<int>) returns (s: int)
  requires n >= 0
  ensures s == (Σi : 0 <= i < n : V[i])
{
  s := sumaVectorGen(V,  0 ,  0 );   Valores iniciales de j y sacum
}

Implementación de la función inmersora

method sumaVectorGen(V[0..n): array<int>, j: int, sacum: int) returns (s: int)
  requires n >= 0 \({\wedge}\) 0 <= j <= n \({\wedge}\) sacum == (Σi : 0 <= i < j : V[i])
  ensures s == (Σi : 0 <= i < n : V[i])
  decreases n - j
{
  if j == n then {
    s := sacum;
  } else {
    s := sumaVectorGen(V, j + 1, sacum + V[j]);
  }
}

ºquí explicarlo intuitivamente con dibujo. sacum contiene lo acumulado.

Corrección

  1. Condición de caso base bien definida:

    n >= 0 \({\wedge}\) 0 <= j <= n \({\wedge}\) sacum == (Σi : 0 <= i < j : V[i])  ==>  true

  2. Caso base correcto:

       n >= 0 \({\wedge}\) 0 <= j <= n \({\wedge}\) sacum == (Σi : 0 <= i < j : V[i]) \({\wedge}\) j == n
    ==>
       sacum == (Σi : 0 <= i < n : V[i])

  3. Precondición en llamada recursiva:

       n >= 0 \({\wedge}\) 0 <= j <= n \({\wedge}\) sacum == (Σi : 0 <= i < j : V[i]) \({\wedge}\) j != n
    ==>
       n >= 0 \({\wedge}\) 0 <= j + 1 <= n \({\wedge}\) sacum + V[j] == (Σi : 0 <= i < j + 1 : V[i])

  1. Caso recursivo correcto:

    n >= 0 \({\wedge}\) 0 <= j <= n \({\wedge}\) sacum == (Σi : 0 <= i < j : V[i]) \({\wedge}\) j != n
      \({\wedge}\) s == (Σi : 0 <= i < n : V[i])
 ==>
    s == (Σi : 0 <= i < n : V[i])

  1. Función de cota ≥ 0:

    n >= 0 \({\wedge}\) 0 <= j <= n \({\wedge}\) sacum == (Σi : 0 <= i < j : V[i])
 ==>
    n - j >= 0

  1. Función de cota decrece:

    n >= 0 \({\wedge}\) 0 <= j <= n \({\wedge}\) sacum == (Σi : 0 <= i < j : V[i]) \({\wedge}\) j != n
 ==>
    n - (j + 1) >= n - j

Análisis de coste

Coste en tiempo de sumaVectorGen: Θ(n - j)

Coste en tiempo de sumaVector: Θ(n), donde n es la longitud del vector pasado como parámetro.

3.6Técnica de desplegado y plegado

Técnica de desplegado y plegado

Sirve para obtener una función recursiva final a partir de una implementación recursiva no final.

Dada una función f recursiva no final:

  1. Definir una generalización fGen en función de f.
  2. Desplegar f dentro de fGen.
  3. Realizar transformaciones elementales en la definición de fGen.
  4. Plegar fGen.

Tras estos pasos, se obtiene una definición de fGen recursiva final.

Ejemplo: factorial

Factorial de un número

method factorial(n: int) returns (f: int)
{
  if n == 0 {
    f := 1;
  } else {
    f’ := factorial(n - 1);
    f := f’ * n;
  }
}

Para facilitar el razonamiento, representamos factorial de manera funcional:

factorial(n) == if n == 0 then 1 else n * factorial(n - 1)

Paso 1. Definir una generalización

Definimos factorialGen del siguiente modo:

factorialGen(n, s) == factorial(n) * s

Sabemos que factorialGen es una generalización de factorial, porque la primera se comporta igual que la segunda en el caso en el que s == 1.

factorialGen(n, 1) == factorial(n) * 1 == factorial(n)

Paso 2. Desplegar la definición de factorial

factorialGen(n, s) == factorial(n) * s

factorial(n) == if n == 0 then 1 else n * factorial(n - 1)

   factorialGen(n, s)
==
   factorial(n) * s
==
   (if n == 0 then 1 else n * factorial(n - 1)) * s

Paso 3. Transformaciones elementales en factorialGen

factorialGen(n, s) == factorial(n) * s

factorial(n) == if n == 0 then 1 else n * factorial(n - 1)

   factorialGen(n, s)
==
   factorial(n) * s
==
   (if n == 0 then 1 else n * factorial(n - 1)) * s
==
   if n == 0 then 1 * s else n * factorial(n - 1) * s
==
   if n == 0 then s else factorial(n - 1) * (n * s)

Paso 4. Plegado de factorialGen

factorialGen(n, s) == factorial(n) * s

factorial(n) == if n == 0 then 1 else n * factorial(n - 1)

   factorialGen(n, s)
==
   factorial(n) * s
==
   (if n == 0 then 1 else n * factorial(n - 1)) * s
==
   if n == 0 then 1 * s else n * factorial(n - 1) * s
==
   if n == 0 then s else factorial(n - 1) * (n * s)
==
   if n == 0 then s else factorialGen(n - 1, n * s)     Definición recursiva final

Resultado final

La función factorialGen es recursiva final.

method factorialGen(n: int, s: int) returns (f: int)
{
  if n == 0 {
    f := s;
  } else {
    f := factorialGen(n - 1, n * s);
  }
}

Implementamos factorial como caso particular de factorialGen:

method factorial(n: int) returns (f: int)
{
  f := factorialGen(n, 1);
}
