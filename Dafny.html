La plataforma Dafny

  • Dafny es:

    • Un lenguaje de programación,
    • Una herramienta para comprobar la corrección de los programas escritos en el lenguaje de programación.

  • El código Dafny verificado puede traducirse a múltiples lenguajes:

    • C#
    • Java
    • JavaScript
    • Go
    • Python

https://dafny.org/

Ejemplo 1:
División entera

Especificación en Dafny

method divide(a: int, b: int) returns (q: int, r: int)
  requires a >= 0 && b > 0
  ensures a == b * q + r && 0 <= r < b
  • Dafny proporciona un tipo nat para los números naturales:
method divide(a: nat, b: nat) returns (q: nat, r: nat)
  requires b > 0
  ensures a == b * q + r && r < b

Implementación en Dafny

method divide(a: nat, b: nat) returns (q: nat, r: nat)
  requires b > 0
  ensures a == b * q + r && r < b
{
  q := 0; r := a;
  while r >= b
    invariant a == b * q + r
  {
    q := q + 1;
    r := r - b;
  }
}
  • Ver qué pasa si se cambian algunas asignaciones
  • Decir que se puede añadir un decreases

Ejemplo 2:
Multiplicar por dos un vector

Multiplicar por dos un vector

  • Queremos hacer un método que reciba un vector (como parámetro de E/S) de números enteros y lo transforme multiplicando cada uno de sus elementos por 2.

  • En el lenguaje pseudocódigo utilizado por nosotros:

method multPorDos(inout V[0..n): array<int>)
  requires n >= 0 \({\wedge}\) V == Vold
  ensures (\(\forall\)j : 0 <= j < n : V[j] == Vold[j] * 2)

Diferencias en Dafny

  • No se pueden especificar parámetros de E/S.

    • Los parámetros de tipos básicos: (int, bool, etc.) son inmutables.
    • Los parámetros de tipos no básicos: (array, objetos) pueden ser modificados.

  • La longitud de un vector V se indica mediante V.Length, en lugar de utilizar una variable aparte.

  • Sintaxis de los cuantificadores:

    forall i | P :: Q   // en lugar de (∀i : P : Q)
exists i | P :: Q   // en lugar de (∃i : P : Q)

  • En la postcondición podemos utilizar old(expr) para hacer referencia a los valores de expr antes de ejecutar el método.

    • Sirve como alternativa a las igualdades V == Vold de la precondición.

Especificación en Dafny

method multDos(V: array<int>)
  requires V.Length >= 0
  ensures forall j | 0 <= j < V.Length :: V[j] == 2 * old(V[j])

Como V.Length es de tipo nat podemos omitir la precondición:

method multDos(V: array<int>)
  ensures forall j | 0 <= j < V.Length :: V[j] == 2 * old(V[j])

Además, si el método va a modificar el array pasado como parámetro, es necesario indicarlo en la especificación:

method multDos(V: array<int>)
  ensures forall j | 0 <= j < V.Length :: V[j] == 2 * old(V[j])
  modifies V

Implementación en Dafny

method multDos(V: array<int>)
  ensures forall j | 0 <= j < V.Length :: V[j] == 2 * old(V[j])
  modifies V
{
  var i := 0;
  while i != V.Length
    invariant 0 <= i <= V.Length 
              && (forall j | 0 <= j < i :: V[j] == old(V[j]) * 2)
              && (forall j | i <= j < V.Length :: V[j] == old(V[j]))
  {
    V[i] := V[i] * 2;
    i := i + 1;
  }
}

Ejemplo 3:
Sumar los elementos de un vector

Sumar los elementos de un vector

En nuestro lenguaje pseudocódigo:

method sumaVector(V[0..n): array<int>) returns (s: int)
requires n >= 0
ensures s == (Σj : 0 <= j < n : V[j])

  • En Dafny no existen los cuantificadores Σ, ∏, #, máx, etc.

  • No obstante, podemos definir funciones propias para incorporar a las fórmulas lógicas.

Funciones fantasma (ghost functions)

  • Una función fantasma es una definición de función que solamente sirve a efectos de especificación.

    • Es decir no se genera código ejecutable para funciones fantasma.

  • Para definir funciones, Dafny utiliza una sintaxis funcional que utiliza recursión en lugar de bucles.

  • Por ejemplo, definimos recursivamente suma(V, i, j) como la suma de los elementos del segmento V[i..j) del vector V.

    \[ \mathit{suma}(V, i, j) = \begin{cases} 0 & \text{si }i = j \\ V[i] + \mathit{suma}(V, i + 1, j) & \text{si }i < j \end{cases} \]

Funciones fantasma

ghost function suma(V: array<int>, i: int, j: int): int
  requires 0 <= i <= j <= V.Length
  decreases j - i
  reads V
{
    if i == j then 0 else V[i] + suma(V, i + 1, j)
}

Por tanto, la especificación de nuestro método en Dafny queda del siguiente modo:

method sumaVector(V: array<int>) returns (s: int)
  ensures s == suma(V, 0, V.Length)

Posibles implementaciones

Recorriendo el vector de derecha a izquierda: 👍🏻

method sumaVector(V: array<int>) returns (s: int)
  ensures s == suma(V, 0, V.Length)
{
  s := 0;
  var i := V.Length;
  while i > 0
    invariant 0 <= i <= V.Length && s == suma(V, i, V.Length)
  {
    s := s + V[i - 1];
    i := i - 1;
  }
}

Recorriendo el vector de izquierda a derecha: 👎🏻

method sumaVector(V: array<int>) returns (s: int)
  ensures s == suma(V, 0, V.Length)
{
  s := 0;
  var i := 0;
  while i < V.Length
    invariant 0 <= i <= V.Length && s == suma(V, 0, i)   ❌
  {
    s := s + V[i];
    i := i + 1;
  }
}

Sentencias assert

  • Nos permiten determinar si una determinada fórmula se cumple o no en un momento determinado.

  • Además de servir para depurar, permiten guiar a Dafny en el proceso de verificación.

while i > 0
  invariant 0 <= i <= V.Length && s == suma(V, 0, i)   
{
  s := s + V[i];
  assert s == suma(V, 0, i + 1); ❌
  i := i + 1;
}

Dafny no puede demostrar que suma(V, 0, i) + V[i] == suma(V, 0, i + 1)

\[ \mathit{suma}(V, i, j) = \begin{cases} 0 & \text{si }i = j \\ V[i] + \mathit{suma}(V, i + 1, j) & \text{si }i < j \end{cases} \]

Lema auxiliar

Lema

Para todo \(V~[0..n)\), \(i\), \(j\) tal que \(0 \leq i \leq j < n\) se cumple:

\[ \mathit{suma}(V, i, j) + V[j] = \mathit{suma}(V, i, j + 1) \]

En Dafny:

lemma sumaPorLaDerecha(V: array<int>, i: int, j: int)
  requires 0 <= i <= j < V.Length
  ensures suma(V, i, j) + V[j] == suma(V, i, j + 1)

Demostración

Por inducción sobre \(j - i\):

Base (\(i = j\)):

\[ \mathit{suma}(V, j, j) + V[j] = V[j] = \mathit{suma}(V, j, j + 1) \]

Paso inductivo: (\(i < j\))

\[ \begin{array}{ll} & \mathit{suma}(V, i, j) + V[j] \\ = & \qquad \{\text{definición de $\mathit{suma}$}\}\\ & V[i] + \mathit{suma}(V, i + 1, j) + V[j] \\ = & \qquad \{\text{hipótesis de inducción}\}\\ & V[i] + \mathit{suma}(V, i + 1, j + 1) \\ = & \qquad \{\text{definición de $\mathit{suma}$}\}\\ & \mathit{suma}(V, i, j + 1) \\ \end{array} \]

Demostración en Dafny

lemma sumaPorLaDerecha(V: array<int>, i: int, j: int)
  requires 0 <= i <= j < V.Length
  ensures suma(V, i, j + 1) == suma(V, i, j) + V[j]
  decreases j - i
{
  if i == j {
    assert suma(V, i, j + 1) == V[j] == suma(V, i, j) + V[j];
  } else {
    calc {
        suma(V, i, j + 1);
    ==
        V[i] + suma(V, i + 1, j + 1);
    == { sumaPorLaDerecha(V, i + 1, j); }
        V[i] + suma(V, i + 1, j) + V[j];
    ==
        suma(V, i, j) + V[j];
    }
  }
}

Uso del lema

while i > 0
  invariant 0 <= i <= V.Length && s == suma(V, 0, i)   
{
  s := s + V[i];
  assert s == suma(V, 0, i + 1) by { sumaPorLaDerecha(V, 0, i); }  ✅
  i := i + 1;
}
