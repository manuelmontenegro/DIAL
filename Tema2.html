¿En qué consiste derivar un algoritmo?

  • Consiste en construir el algoritmo a partir de su especificación, verificándolo a medida que se construye.

  • El proceso suele estar dirigido por la postcondición. A partir de ella puede determinarse:

    • Si es necesario un bucle para el algoritmo.

    • En caso de ser necesario, cuál es su invariante y condición de salida.

  • Proceso más o menos sistemático, pero no es mecánico.
  • Muchas decisiones: algunas llevan a un algoritmo, otras llevan a otro, otras no llevan a nada.
    • Es la experiencia la que nos hace intuir
  • No se trata de idea feliz.

2.1Derivación de algoritmos sin bucles

Instrucciones sencillas

  • Si la postcondición contiene igualdades de la forma x == e, podemos intentar asignaciones de la forma x := e.

Ejemplo:

{ true }
???
{ x == 7 }

Si ponemos la expresión x := 7; obtenemos un resultado correcto:

{ true }
x := 7;
{ x == 7 }

wp(x := 7, x == 7)
  === 7 == 7
  === true

Instrucciones basadas en la precondición

  • Si la postcondición contiene igualdades de la forma x == e, pero e no es directamente expresable en el lenguaje de programación, podemos apoyarnos en la precondición.
  • Si la precondición contiene una igualdad de la forma x == e' la asignación debe buscar una forma de transformar e' en e.

{ y >= 0 \({\wedge}\) x == 2y-1 }
???
{ x == 2y }

En este caso, sabemos que 2y == 2 * 2y-1, por lo que podemos probar la asignación x := x * 2;.

{ y >= 0 \({\wedge}\) x == 2y-1 }
x := x * 2;
{ x == 2y }

wp(x := x * 2, x == 2y) === 2 * x == 2y

Comprobamos:

y >= 0 \({\wedge}\) x == 2y-1  ==>  2 * x == 2y   ✅

Instrucciones con distinción de casos

{ P }
x := e;
{ Q }

  • En los casos en los que P no implique wp(x := e, Q), debemos determinar en qué casos no se cumple esa implicación.

  • En muchas ocasiones, esto conllevará una distinción de casos (mediante if).

Ejemplo

{ x >= a }
???
{ x > a \({\wedge}\) x > b }

Si intentamos x := b + 1; tendríamos:

wp(x := b + 1, x > a \({\wedge}\) x > b) === b + 1 > a \({\wedge}\) b + 1 > b

La condición b + 1 > a no se deduce a partir de la precondición.

{ x >= a }
if b + 1 > a { x := b + 1; } else { skip }
{ x > a \({\wedge}\) x > b }

wp((if …), x > a \({\wedge}\) x > b)
  ===   (b + 1 > a  ==> b + 1 > a \({\wedge}\) b + 1 > b)
      \({\wedge}\) (b + 1 <= a ==> x > a \({\wedge}\) x > b)

¿Se deduce a partir de la precondición?

x >= a ==> (
    (b + 1 > a   ==> b + 1 > a \({\wedge}\) b + 1 > b)   ✅
    \({\wedge}\) (b + 1 <= a ==> x > a \({\wedge}\) x > b)          ❌
  )

{ x >= a }
if a < b + 1 then { x := b + 1; } else { x := x + 1; }
{ x > a \({\wedge}\) x > b }

Comprobamos:

x >= a ==> (
    (a < b + 1   ==> b + 1 > a \({\wedge}\) b + 1 > b)   ✅
    \({\wedge}\) (a >= b + 1 ==> x + 1 > a \({\wedge}\) x + 1 > b)  ✅
  )

¿Y si la precondición no nos ayuda?

{ y >= 0 }
???
{ x == 2y }

Necesitaremos un bucle.

Estrategia 1

Si la postcondición es una conjunción (\({\wedge}\)) de fórmulas, escoger la negación de una de ellas como condición del bucle, y el resto como invariante.

Supongamos que la postcondición \(Q\) tiene la forma \(Q_1 \wedge \dots \wedge Q_{n-1} \wedge Q_n\).

Tomamos como invariante \(I \equiv Q_1 \wedge \ldots \wedge Q_{n-1}\) y como condición \(b \equiv \neg Q_n\).

Entonces se cumple la condición de salida del bucle: \(I \wedge \neg b \Rightarrow Q\).

Máximo de un vector: especificación

Derivar una función que, dado un vector no vacío, devuelva el valor máximo del vector.

method maximo(V[0..n): array<int>) returns (m: int)
  requires n >= 1
  ensures m == (máx j : 0 <= j < n : V[j])

A partir de la postcondición, no hay un candidato claro para la condición del bucle.

Estrategia 2

Reemplazar una constante k que aparezca en la poscondición por una variable nueva i, y añadir la condición i == k.
