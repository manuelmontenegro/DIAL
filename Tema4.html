4.1Esquema general

¿Qué es Divide y Vencerás?

  • Es una técnica algorítmica que consiste en resolver un problema mediante su descomposición en problemas más pequeños, pero del mismo tipo.

  • Las soluciones de cada subproblema se combinan para construir una solución al problema original.

¿Qué es Divide y Vencerás?

Los casos base

  • Son aquellos en los que el problema no se subdivide más, y se resuelve de manera directa, o bien utilizando otro algoritmo distinto.

Esquema general

method divideYVencerás(x̅: problema) returns (y̅: solución)
  decreases tamaño(x̅)
{
  if casoSencillo(x̅) {
    y̅ := metodoDirecto(x̅);
  } else {
    x̅1, …, x̅n := descomponer(x̅);    Descomposición del problema original

    y̅1 := divideYVencerás(x̅1);      Resolución de subproblemas
    …
    y̅n := divideYVencerás(x̅n);

    y̅ := combinar(x̅, y̅1, …, y̅n)     Combinación de resultados
  }
}

Implementación en C++

Cabecera de la función:

int maximo(const vector<int> &v);

Añadimos parámetros de inmersión que delimitan el segmento v[c..f) del vector sobre el cual calcular el máximo:

int maximo(const vector<int> &v, int c, int f) {
  assert (0 <= c && f <= n && f - c >= 1);      // Precondición: segmento no vacío

  if (f - c == 1) {     // Caso base: segmento unitario
    return v[c];
  } else {              // Caso recursivo
    int m = (c + f) / 2;
    int max1 = maximo(v, c, m);
    int max2 = maximo(v, m, f);
    return max(max1, max2);
  }
}

Análisis de coste

Si \(m\) denota la diferencia f - c:

\[ T(m) = \begin{cases} k_1 & \text{si } m = 1 \\ 2T\left(\frac{m}{2}\right) + k_2 & \text{si } m > 1 \\ \end{cases} \]

Utilizando la regla de la división: \(a = 2\), \(b = 2\), \(d = 0\). Por tanto, \(a > b^d\):

\[ T(m) \in \Theta(m^{\log_2 2}) = \Theta(m) = \Theta(\texttt{f} - \texttt{c}) \]

Llamada inicial

int maximo(const vector<int> &v) {
  return maximo(v, 0, v.size());
}

Coste: \(\Theta(\texttt{f} - \texttt{c}) = \Theta(n - 0) = \Theta(n)\), donde \(n = \texttt{v.size()}\).

Generalización

Elementos de tipo T, en lugar de tipo int:

template <typename T>
T maximo(const vector<T> &v, int c, int f) {
  if (c + 1 == f) {
    return v[c];
  } else {
    int m = (c + f) / 2;

    T max1 = maximo(v, c, m);
    T max2 = maximo(v, m, f);

    return max(max1, max2);
  }
}

template <typename T>
T maximo(const vector<T> &v) {
  return maximo(v, 0, v.size());
}

cout << maximo<int>({4, 5, 1, 3, -3, 6});   // Imprime: 6

method BusquedaBinaria(V[0..n): array<int>, x: int) returns (k: int)
{
  k := BusquedaBinariaGen(V, x, 0, n);

Búsqueda binaria: caso base

ensures  c <= k <= f \({\wedge}\) (\(\forall\)i : c <= i < k : V[i] < x) \({\wedge}\) (\(\forall\)i : k <= i < f : V[i] >= x)

Cuando c == f, la única posibilidad es k := c, que satisface la postcondición.

method BusquedaBinariaGen(V[0..n): array<int>, x: int, c: int, f: int) returns (k: int)
  requires (\(\forall\)i, j : 0 <= i <= j < n : V[i] <= V[j]) \({\wedge}\) 0 <= c <= f <= n
  ensures  c <= k <= f \({\wedge}\) (\(\forall\)i : c <= i < k : V[i] < x) \({\wedge}\) (\(\forall\)i : k <= i < f : V[i] >= x)
{
  if c == f then {
    k := c;     Caso base
  } else {
    …
  }
}

Búsqueda binaria: caso recursivo

  • Calculamos el punto intermedio m entre c y f.

  • Si V[m] < x, podemos descartar el segmento V[c..m] del vector.

  • Si V[m] >= x, podemos descartar el segmento V[m+1..f) del vector.

Búsqueda binaria: caso recursivo

method BusquedaBinariaGen(V[0..n): array<int>, x: int, c: int, f: int) returns (k: int)
  requires (\(\forall\)i, j : 0 <= i <= j < n : V[i] <= V[j]) \({\wedge}\) 0 <= c <= f <= n
  ensures  c <= k <= f \({\wedge}\) (\(\forall\)i : c <= i < k : V[i] < x) \({\wedge}\) (\(\forall\)i : k <= i < f : V[i] >= x)
  decreases f - c
{
  var m: int;
  if c == f then {
    k := c;
  } else {
    m := (c + f) div 2;
    if V[m] < x then {
      k := BusquedaBinariaGen(V, x, m + 1, f);
    } else {
      k := BusquedaBinariaGen(V, x, c, m);
    }
  }
}

Análisis de coste

Si \(m\) denota la diferencia f - c:

\[ T(m) = \begin{cases} k_1 & \text{si } m = 0 \\ T\left(\frac{m}{2}\right) + k_2 & \text{si } m > 0 \end{cases} \qquad \Rightarrow \qquad T(m) \in \Theta(\log~m) \]

Coste en tiempo de BusquedaBinariaGen:

\[ \Theta(\log(\texttt{f - c})) \]

Coste en tiempo de BusquedaBinaria:

\[ \Theta(\log~\texttt{n}) \]

donde n es el tamaño del vector.

Implementación en C++

template <typename T>
int busqueda_binaria(const vector<T> &v, const T &x, int c, int f) {
  if (c == f) {
    return c;
  } else {
    int m = (c + f) / 2;
    if (v[m] < x) {
      return busqueda_binaria(v, x, m + 1, f);
    } else {
      return busqueda_binaria(v, x, c, m);
    }
  }
}

template <typename T>
int busqueda_binaria(const vector<T> &v, const T &x) {
  return busqueda_binaria(v, x, 0, v.size());
}

4.3Búsqueda binaria en el espacio de soluciones

Ejemplo: Cola de pacientes

  • En una cola de pacientes, conocemos el tiempo que necesita cada paciente en ser atendido.

  • Tenemos varios médicos. El turno de cada médico dura una cantidad de minutos.

    • Si el tiempo restante en el turno del médico es igual o mayor que la cantidad que necesita el siguiente paciente, el médico atiende al paciente.
    • En caso contrario, el médico se va y llega otro médico, que atenderá al siguiente paciente.

  • Supongamos que tenemos 3 médicos y que el turno de cada uno de ellos dura 30’.

    • El primer médico atiende a los dos primeros pacientes (15’ + 10’ = 25’). Como no le queda tiempo para atender al tercero, se va.
    • El segundo médico atiende a los dos siguientes pacientes (10’ + 17’ = 27’). Como no le queda tiempo para atender al que va después, se va.
    • El tercer médico solo puede atender al paciente de 12’. El último paciente se queda sin atender.

❌ Un turno de 30’ no es suficiente para atender a todos los pacientes

  • ¿Y si, con la misma cantidad de médicos, la duración del turno fuera de 40’?

    • El primer médico atiende a los tres primeros pacientes (15’ + 10’ + 10’ = 35’). Como no le queda tiempo para atender al cuarto, se va.
    • El segundo médico atiende a los dos siguientes pacientes (17’ + 12’ = 29’). Como no le queda tiempo para atender al último, se va.
    • El tercer médico atiende al último paciente (20’).

✅ Un turno de 40’ sí es suficiente para atender a todos los pacientes

Descripción del problema

  • Dado un vector V[0..n) con los tiempos que requieren los pacientes.

  • Dado un número k de médicos.

¿Cuánto ha de durar, como mínimo, el turno de todos los médicos para que todos los pacientes puedan ser atendidos?

Observaciones

  • La solución es una cantidad entera de minutos (un valor de tipo int).

  • Espacio de soluciones: cualquier número entero entre 20 y 84.

    • ¿Por qué el mínimo es 20? Porque el paciente que más tiempo necesita en ser atendido requiere 20’. Si el turno de cada médico dura menos, ese paciente nunca podrá ser atendido.

    • ¿Por qué el máximo es 84? Porque es la suma de los tiempos que necesita un paciente. Si el turno de un médico durase 84’, él solo podría atender a todos los pacientes.

Observaciones

  • Si es suficiente un turno de t minutos, también lo será uno de mayor duración.
  • Si no es suficiente un turno de t minutos, tampoco lo será uno de menor duración.

Por tanto, el espacio de soluciones tiene esta forma:

  • ✅= Todos los pacientes son atendidos con esa cantidad de minutos por turno.
  • ❌= Se quedan pacientes sin atender con esa cantidad de minutos por turno.

Especificación

  • Suponemos un predicado sePuede(V[0..n), k, t) que se evalúe a true si y solo si una duración t de cada turno es suficiente para que k médicos atiendan a los n pacientes.

  • Estamos interesados en implementar la siguiente función:

    method turnoMinimo(V[0..n): array<int>, k: int) returns (t: int)
      requires …
      ensures sePuede(V, k, t) \({\wedge}\) (\(\forall\)t’ : t’ < t : \(\neg\)sePuede(V, k, t’))

Condiciones que ha de cumplir sePuede

  • El predicado sePuede ha de cumplir dos propiedades:

sePuede(V, k, t) ==> (\(\forall\)t’ : t’ > t : sePuede(V, k, t’))

\(\neg\)sePuede(V, k, t) ==> (\(\forall\)t’ : t’ < t : \(\neg\)sePuede(V, k, t’))

Inmersión de dos parámetros

Realizamos inmersión con dos parámetros, que determinan los límites del espacio de búsqueda:

method turnoMinimoGen(V[0..n): array<int>, k: int, c: int, f: int) returns (t: int)
  requires … \({\wedge}\) c <= f \({\wedge}\) sePuede(V, k, f)
  ensures c <= t <= f \({\wedge}\) sePuede(V, k, t) \({\wedge}\) (\(\forall\)t’ : c <= t’ < t : \(\neg\)sePuede(V, k, t’))

La función turnoMinimo se implementa del siguiente modo:

method turnoMinimo(V[0..n): array<int>, k: int) returns (t: int)
{
  var m: int := maximo(V);
  var s: int := suma(V);
  t := turnoMinimoGen(V, k, m, s);
}

Búsqueda binaria en espacio de soluciones

  • m = punto medio entre c y f

  • Caso 1: Es posible atender a los pacientes en tiempo m. Buscamos en [c, m].

  • Caso 2: No es posible atender a los pacientes en tiempo m. Buscamos en [m+1, f].

Búsqueda binaria en espacio de soluciones

  • Caso base: c == f. Solución: c

Implementación

method turnoMinimoGen(V[0..n): array<int>, k: int, c: int, f: int) returns (t: int)
  requires …
  ensures …
  decreases f - c
{
  if c == f {
    t := c;
  } else {
    var m: int := (c + f) div 2;
    if sePuede(V, k, m) {
      t := turnoMinimoGen(V, k, c, m);
    } else {
      t := turnoMinimoGen(V, k, m + 1, f);
    }
  }
}

Método auxiliar: sePuede

method sePuede(V[0..n): array<int>, k: int, t: int) returns (b: bool)
{
  var i: int, tdisp: int, numMedicos: int;
  i := 0; tdisp := t; numMedicos := 1;
  while i < n {
    if V[i] > tdisp {
      numMedicos := numMedicos + 1;
      tdisp := t - V[i];
    } else {
      tdisp := tdisp - V[i];
    }
    i := i + 1;
  }

  b := (numMedicos <= k);
}

Coste en tiempo: Θ(n)

Análisis de coste de turnoMinimoGen

Si \(m = \texttt{f} - \texttt{c}\) y \(n\) es la longitud del vector, tenemos:

\[ T(n, m) = \begin{cases} k_1 & \text{si } m = 0 \\ k_2n + T\left(n, \frac{m}{2}\right) & \text{si } m > 0 \\ \end{cases} \]

\[ T(n, m) \in \Theta(n \log m) \]

El coste de la llamada inicial es \(\Theta(n \log M)\), donde \(M\) es la longitud del espacio de soluciones.

\[ M = \left(\sum_{i=0}^{n-1} V[i]\right) - \max \{ V[i] \mid {0 \leq i < n} \} \]

4.4Ordenación rápida (Quicksort)

Resumen del algoritmo

Supongamos que queremos ordenar el siguiente vector:

5 4 2 1 8 3 7 6

Escogemos un elemento arbitrario (por ejemplo, el primero), y lo denominamos pivote:

pivote
5 4 2 1 8 3 7 6

Suponemos que existe un algoritmo de partición que desplaza todos los elementos menores que el pivote a la izquierda de él.

Ordenamos (mediante una llamada recursiva) el segmento del vector que está a la izquierda del pivote:

1 2 3 4 5 8 7 6

Ordenamos (mediante una llamada recursiva) el segmento del vector que está a la derecha del pivote:

menores que pivote pivote mayores que pivote
1 2 3 4 5 6 7 8

El vector queda ordenado.

Algoritmo de partición

  • Dado un vector y un pivote, el algoritmo de partición reordena los elementos del vector para que quede dividido en tres regiones:
    • Elementos que son menores que el pivote: V[0..p)
    • Elementos que son iguales al pivote: V[p..q)
    • Elementos que son mayores que el pivote: V[q..n)
0 p q n
< pivote = pivote > pivote

El algoritmo, además de mover los elementos, devuelve el par (p, q) con las posiciones que delimitan cada zona.

Partición sobre un segmento del vector

  • Generalizamos el algoritmo de partición para que solamente afecte a un segmento V[c..f) del vector V de entrada.
c p q f
< pivote = pivote > pivote

 

Especificación:

method partition(inout V[0..n): array<int>, c: int, f: int, pivote: int) returns (p: int, q: int)
  requires n >= 0 \({\wedge}\) 0 <= c <= f <= n
  ensures c <= p <= q <= f \({\wedge}\)
          (\(\forall\)i : c <= i < p : V[i] < pivote) \({\wedge}\)
          (\(\forall\)i : p <= i < q : V[i] == pivote) \({\wedge}\)
          (\(\forall\)i : q <= i < f : V[i] > pivote)

Funcionamiento del algoritmo

  • El algoritmo de partición es iterativo.

  • Recorre los elementos del segmento V[c..f) de izquierda a derecha.

  • Además de las variables de salida (p y q), usa otra variable más (j), que también denota una posición del vector.

  • Invariante del bucle de recorrido:

    c p j q f
    < pivote = pivote ??? > pivote
  • El segmento V[c..p) contiene los elementos ya recorridos menores que el pivote.
  • El segmento V[p..j) contiene los elementos ya recorridos iguales al pivote.
  • El segmento V[j..q) contiene elementos que aún no han sido recorridos.
  • El segmento V[q..f) contiene los elementos ya recorridos mayores que el pivote.

pair<int, int> partition(vector<T> &V, int c, int f, const T &pivote) {

template <typename T>
pair<int, int> partition(vector<T> &V, int c, int f, const T &pivote) {
  int p = c, j = c, q = f;
  while (j != q) {
    if (V[j] == pivote) {
      j++;
    } else if (V[j] < pivote) {
      swap(V[p], V[j]);
      p++; j++;
    } else {
      ...
    }
  }

Implementación en C++

template <typename T>
pair<int, int> partition(vector<T> &V, int c, int f, const T &pivote) {
  int p = c, j = c, q = f;
  while (j != q) {
    if (V[j] == pivote) {
      j++;
    } else if (V[j] < pivote) {
      swap(V[p], V[j]);
      p++; j++;
    } else {
      swap(V[j], V[q - 1]);
      q--;
    }
  }
  return {p, q};
}

Coste de partition

  • Función de cota: q - j.
  • Inicialmente, q = f y j = c, por lo que el bucle hace f - c iteraciones.
  • En cada iteración se realizan intercambios e incrementos y comparaciones con el pivote, que son operaciones de coste constante.

Coste en tiempo: Θ(f - c)

4.5Ordenación por mezclas (Mergesort)

Resumen del algoritmo

Partimos del siguiente vector:

5 13 24 20 7 9 10 8 21

Dividimos el vector en dos mitades:

5 13 24 20 7
9 10 8 21

Ordenamos recursivamente cada mitad:

5 7 13 20 24
8 9 10 21

Fusionamos las mitades ordenadas en un único vector:

5 7 8 9 10 13 20 21 24

    copy(V.begin() + m, V.end(), W.begin());

Algoritmo de mezcla (merge)

Algoritmo de mezcla

  • Recibe dos vectores U[0..n) y W[0..m) como parámetros de entrada y un vector V[0..q) como parámetro de salida.

    • Se cumple que n + m == q.
  • Al final del algoritmo, el vector de salida V contendrá los elementos de U y W, y además estará ordenado.

Código C++

template <typename T>
void merge(const vector<T> &U, const vector<T> &W, vector<T> &V) {
  // Inicialización
  int i = 0, j = 0, k = 0;

  // Mezcla, mientras queden elementos por colocar en ambos vectores
  while (i < U.size() && j < W.size()) {
    if (U[i] <= W[j]) {
      V[k] = U[i];
      i++; k++;
    } else {
      V[k] = W[j];
      j++; k++;
    }
  }

  // Cuando uno de los vectores de entrada se agota, copiamos lo que queda del otro a V
  copy(U.begin() + i, U.end(), V.begin() + k);
  copy(W.begin() + j, W.end(), V.begin() + k);
}

Análisis de coste de merge

Caso 1: salimos del bucle cuando i == n

  • El primer bucle ha hecho n + j iteraciones (coste Θ(1) cada una).

  • La copia de W[j..m) a V realiza m - j iteraciones (coste Θ(1) cada una).

  • Total: (n + j) + (m - j) = n + m iteraciones.

Caso 2: salimos del bucle cuando j == m

  • El primer bucle ha hecho m + i iteraciones (coste Θ(1) cada una).

  • La copia de U[i..n) a V realiza n - i iteraciones (coste Θ(1) cada una).

  • Total: (m + i) + (n - i) = n + m iteraciones.

Coste: Θ(n + m)

Da igual el vector que se me gaste, porque luego tengo que recorrer lo que me queda por gastar del otro.

4.6Par de puntos más cercano

El problema

  • Tenemos n puntos en el plano.

  • ¿Cuáles son los dos puntos más cercanos entre sí?

Algoritmo de fuerza bruta

Definiciones de tipos de datos

  • Tipo de datos Punto:

    struct Punto {
      double x, y;
    };

  • Tipos de datos Solucion:

    struct Solucion {
      Punto p1, p2;
      double distancia;
    };  

  • Función que calcula la distancia entre dos puntos:

    double distancia(const Punto &p1, const Punto &p2) {
      return sqrt((p1.x - p2.x) * (p1.x - p2.x) + (p1.y - p2.y) * (p1.y - p2.y));
    }

Algoritmo de fuerza bruta

Solucion fuerza_bruta(const std::vector<Punto> &puntos) {
  assert(puntos.size() >= 2);

  double dist_minima = numeric_limits<double>::infinity();
  Solucion sol;

  for (int i = 0; i < puntos.size(); ++i) {
    for (int j = i + 1; j < puntos.size(); ++j) {
      double d = distancia(puntos[i], puntos[j]);
      if (d < dist_minima) {
        dist_minima = d;
        sol = {puntos[i], puntos[j], d};
      }
    }
  }

  return sol;
}

Coste: \(\Theta(n^2)\), donde \(n\) es la longitud del vector puntos.

Algoritmo Divide y Vencerás
(1º intento)

Algoritmo Divide y Vencerás

  • Dividir el plano en dos semiplanos con la misma cantidad de puntos.

  • Requiere ordenar los puntos por coordenada \(x\)

    • Coste \(\Theta(n \log n)\).

  • Resolver el problema recursivamente en cada semiplano.

Algoritmo Divide y Vencerás

  • Sean \(d_1\) y \(d_2\) las distancias mínimas entre puntos del semiplano izquierdo y el semiplano derecho.

  • Sea \(d = \min(d_1, d_2)\)

  • No estamos considerando los pares de puntos que pertenecen a semiplanos distintos.

  • ¿Existe un par de puntos \(P_1\) y \(P_2\), cada uno en un semiplano que estén a distancia menor que \(d\)?

Franja central

  • Comparamos solamente los puntos que están que están a distancia \(<d\) de la línea divisoria.

  • Estos puntos están en una franja centrada en la línea divisoria y de ancho \(2d\).

Llamada inicial al algoritmo DyV

  • La función divide_y_venceras tiene dos parámetros de inmersión, que indican el segmento puntos[c..f) el vector a considerar:

    Solucion divide_y_venceras(const vector<Punto> &puntos, int c, int f);

  • La llamada inicial se hace con la totalidad del vector, y habiendo ordenados los puntos previamente:

    Solucion par_puntos_cercano(vector<Punto> &puntos) {
      sort(puntos.begin(), puntos.end(), comparar_x); Ordena el vector puntos por coordenada \(x\)
      return divide_y_venceras(puntos, 0, puntos.size());  Llamada inicial
    }

    donde comparar_x se ha definido del siguiente modo:

    bool comparar_x(const Punto &p1, const Punto &p2) { return p1.x < p2.x; }

Algoritmo divide y vencerás

Solucion divide_y_venceras(const vector<Punto> &puntos, int c, int f) {
  if (f - c <= 3) {
    return fuerza_bruta(puntos, c, f);
  } else {
    int m = (c + f) / 2; Mitad del segmento puntos[c..f)
    double frontera_x = puntos[m].x;

    Solucion sol1 = divide_y_venceras(puntos, c, m);                 Llamada recursiva semiplano izquierdo
    Solucion sol2 = divide_y_venceras(puntos, m, f);                 Llamada recursiva semiplano derecho
    Solucion sol = (sol1.distancia < sol2.distancia) ? sol1 : sol2;  Nos quedamos con la menor distancia

    vector<Punto> franja;
    for (int i = c; i < f; i++) {                                  Obtenemos los puntos de la franja central
      if (abs(puntos[i].x - frontera_x) < sol.distancia) franja.push_back(puntos[i]);
    }

    if (franja.size() >= 2) {
      Solucion sol_franja = fuerza_bruta(franja);       Obtenemos la distancia mínima entre puntos de la franja
      return sol_franja.distancia < sol.distancia ? sol_franja : sol;    
    } else {
      return sol;
    }
  }
}

Algoritmo Divide y Vencerás
(2º intento)

Distancia mínima entre puntos dentro de la franja

  • Ordenamos los puntos por coordenada y, de manera creciente.

  • Para cada punto \(P(x, y)\)

    • Comparamos el punto \(P\) con todos los puntos \((x',y')\) tales que \(y' \in [y, y + d)\)

Solucion par_puntos_franja(vector<Punto> &franja, double d) {
  assert(franja.size() >= 2);

  sort(franja.begin(), franja.end(), comparar_y);       Ordenamos por coordenada \(y\)

  Solucion sol = { franja[0], franja[1], distancia(franja[0], franja[1]) };
 
  for (int i = 0; i < franja.size(); i++) {
    int j = i + 1;
    while (j < franja.size() && franja[j].y < franja[i].y + d) {
      double dist = distancia(franja[i], franja[j]);
      if (dist < sol.distancia) {
        sol = { franja[i], franja[j], dist };
      }
      j++;
    }
  }

  return sol;
}

4.7Cuándo no utilizar Divide y Vencerás

Cuándo no utilizar Divide y Vencerás

  1. Cuando el problema de tamaño \(n\) se subdivide en problemas de tamaño casi \(n\).

  2. Cuando el problema de tamaño \(n\) se subdivide en casi \(n\) problemas de tamaño fracción.

  3. Cuando el algoritmo DyV tiene coste \(\Theta(n)\) y existe un algoritmo iterativo igual de eficiente.

Subdivisión en problemas de tamaño casi \(n\)

Ejemplo: Fibonacci

long fibonacci(int n) {
  if (n == 0) {
    return 0;
  } else if (n == 1) {
    return 1;
  } else {
    return fibonacci(n - 1) + fibonacci(n - 2);
  }
}

  • El coste viene dado por:

\[ T(n) = \begin{cases} k_1 & \text{si } n \leq 1\\ T(n-1) + T(n-2) + k_2 & \text{si } n > 1\\ \end{cases} \quad \Longrightarrow \quad T(n) \in \Omega(2^\frac{n}{2}) \cap O(2^n) \]

Ejemplo: números combinatorios

\[ {n \choose k} = {n - 1 \choose k - 1} + {n - 1 \choose k} \]

  • Utilizar un algoritmo divide y vencerás para calcular \({n \choose k}\) tiene coste \(O(2^n)\).
  • Solución: utilizar un algoritmo de programación dinámica, con coste \(\Theta(nk)\).

Subdivisión en casi \(n\) problemas

Subdivisión de los algoritmos DyV

\[ T(n) = \begin{cases} k_1 & \text{si } n < n_0\\ a T\left(\frac{n}{b}\right) + k_2 n^d & \text{si } n \geq n_0\\ \end{cases} \]

  • El problema se divide en \(a\) instancias de tamaño \(n/b\) cada una.

  • No es necesario que \(a \leq b\).

  • Pero no podemos tener \(a\) cercanos a \(n\).

    • Por ejemplo, si \(a = n\), \(b = 2\), \(d = 0\), tenemos que \(T(n) \in \Theta(n^{\log n})\)

Algoritmos iterativos más eficientes

Ejemplos en los que no utilizar DyV

  • Sumar los elementos de un vector.

  • Calcular el máximo de un vector.

  • Calcular el segmento de suma máxima.

  • Calcular el segmento más largo que contiene un valor \(k\).
